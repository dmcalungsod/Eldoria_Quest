# Contributing to Eldoria Quest

First off, thank you for considering contributing to Eldoria Quest! It's people like you that make this project great.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.
When you are creating a bug report, please include as many details as possible according to the template provided.

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality. Following these guidelines helps maintainers and the community understand your suggestion and find related suggestions.
When you are creating an enhancement suggestion, please include as many details as possible according to the template provided.

### Pull Requests

The process described here has several goals:

- Maintain Eldoria Quest's quality
- Fix problems that are important to users
- Engage the community in working toward the best possible version of Eldoria Quest

Please follow these steps to have your contribution considered by the maintainers:

1. Ensure your code conforms to our coding standards. Format and lint your code before committing.
2. Update or add relevant documentation if your changes affect user-facing features or system design.
3. Write sufficient tests to cover your new work or reproduce the bug you are fixing. Check that all tests pass.
4. After you submit your pull request, verify that all status checks and continuous integration pipelines are passing.

## Styleguides

### Git Commit Messages

* Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally after the first line

### Coding Standards

* We use standard Python styling conventions (PEP 8).
- Ensure type hints are used consistently across new functions and classes.
- Code should be formatted with tools like `black` and linted with tools like `ruff` or `flake8`.

Thank you for contributing!
