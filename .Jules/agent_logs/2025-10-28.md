## BugHunter Log - 2025-10-28

### 🐛 Issue: Skill Upgrade Cost Bypass
- **Description:** `SkillTrainerView._execute_upgrade` was using the base `upgrade_cost` for all skill upgrades, regardless of the current skill level. This allowed players to upgrade high-level skills for the cheap Level 1 price.
- **Affected Components:** `cogs/skill_trainer_cog.py`
- **Fix:** Updated `_execute_upgrade` to fetch the current skill level from the database and calculate the scaled cost using `get_upgrade_cost(base, level)`.
- **Testing:** Added `tests/test_skill_upgrade_cost.py` to verify the fix. Rewrote `tests/test_skill_exploit.py` to use proper mocks and verify client-side cost manipulation is ignored.
- **Coordination:** @Sentinel @GameForge - The skill upgrade cost logic is now secure against this specific exploit. Please verify any other scaling logic (e.g. item crafting costs) similarly.
