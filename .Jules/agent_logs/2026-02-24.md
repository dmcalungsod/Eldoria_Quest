### ✅ Workflows Added (PR #345)

Five new GitHub Actions workflows were introduced to `.github/workflows/`:

1. **`dependency-review.yml`** — Blocks PRs with moderate+ severity dependency vulnerabilities; posts a summary comment.
2. **`stale.yml`** — Marks issues/PRs stale after 30 days, closes after 7 more; runs daily at 1 AM UTC.
3. **`release.yml`** — Auto-creates GitHub releases with generated changelogs on `v*.*.*` tags.
4. **`codeql.yml`** — Python CodeQL analysis on push/PR to `main`/`master` and weekly (Mon 2 AM UTC). *(Later removed — see below)*
5. **`coverage-comment.yml`** — Runs pytest + coverage against a live MongoDB service on PRs, posts coverage report as PR comment.

No existing workflows (`ci.yml`, `automerge.yml`, `mirror.yml`) were modified.

### ❌ Errors Encountered & Fixes Applied

#### Error 1: Dependency Review – Feature Not Supported
- **Symptom:** `dependency-review` action failed — Dependency graph / GitHub Advanced Security not enabled.
- **Fix (PR #347, merged):** Patched workflow to avoid hard-failing when Advanced Security is unavailable.
- **Action Required:** To fully enable, go to [Security Analysis Settings](https://github.com/dmcalungsod/Eldoria_Quest/settings/security_analysis) and enable **Dependency graph** and **GitHub Advanced Security**.

#### Error 2: CodeQL – Not Supported on Private Repos Without GHAS
- **Symptom:** `codeql-action` requires GitHub Advanced Security, which is not active on this private repo.
- **Fix (PR #348, merged):** `codeql.yml` was removed entirely. CodeQL requires GHAS on private repositories.

#### Error 3: CodeQL – Missing `actions: read` Permission (pre-removal)
- **Symptom:** `codeql-action/analyze@v3` failed to access workflow run context.
- **Fix (commit `e845c80` by dmcalungsod):** Added `actions: read` to the permissions block in `codeql.yml` before its removal.
